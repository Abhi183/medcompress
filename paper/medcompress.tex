\begin{abstract}
Deep learning models for medical image analysis assume GPU hardware that clinic workstations, field laptops, and telemedicine terminals do not have. I present MedCompress, an open-source benchmark that tests how far three compression strategies can push medical imaging models toward CPU-only inference on macOS and Windows. The strategies are quantization-aware training (QAT), knowledge distillation (KD), and a sparse attention mechanism I adapted from Memory Sparse Attention~\citep{chen2026msa}. I evaluate on two clinical tasks: melanoma classification with the ISIC 2020 dataset (33,126 dermoscopy images) and brain tumor segmentation with BraTS 2021 (multi-modal MRI). QAT at INT8 cuts model size 3.8$\times$ with 1.4\% AUC loss on melanoma detection. Distillation from a full U-Net teacher to a lightweight student keeps 94.3\% of teacher Dice on brain tumor segmentation at 7.9$\times$ fewer parameters. Sparse attention with kernel-4 pooling and top-8 routing reduces attention memory 24.5$\times$ at 0.7\% AUC cost. The combined KD + QAT INT8 pipeline yields a 2.7 MB melanoma classifier running in 8.3 ms and a 4.1 MB segmentation model running in 12.6 ms, both on CPU. Code, export pipelines, and a desktop application are released at \url{https://github.com/Abhi183/medcompress}.
\end{abstract}

\section{Introduction}
\label{sec:intro}
% =====================================================================

I manage fleets of Mac and Windows machines for a living. I know what ships through Jamf and SCCM and what does not. A 124 MB segmentation model with GPU dependencies does not. A 4 MB TFLite file does.

Medical imaging models are getting more accurate every year~\citep{gholami2022survey}. The hardware at the point of care has not kept pace. Clinics run five-year-old iMacs. Field offices run Windows laptops with integrated graphics. Telemedicine terminals run whatever procurement found cheapest. Cloud inference addresses the compute gap but creates others: latency on spotty connections, data governance headaches when patient images leave the machine, and vendor lock-in that IT departments have learned to avoid the hard way.

I wanted to find out how far I could compress a medical imaging model before it stops being clinically useful, and whether the result can be deployed the same way I deploy any other endpoint software---pushed through MDM, versioned alongside the app, rolled back with a file swap.

MedCompress is the benchmark I built to answer that. It implements quantization-aware training, knowledge distillation, and sparse attention compression, evaluates each on melanoma classification and brain tumor segmentation, and exports compressed models to TFLite and ONNX for CPU inference. The sparse attention module borrows KV cache pooling and top-k routing from Memory Sparse Attention~\citep{chen2026msa}, a technique originally designed for 100M-token language contexts. I adapted it for Vision Transformer attention in dermoscopy, where it turned out to work surprisingly well (Section~\ref{sec:sparse_results}).

I also built a desktop application that wraps the compressed models in a GUI. A clinician opens an image, clicks a button, and gets a classification or segmentation result, no cloud, no GPU, no Python installation required.

\section{Related Work}
\label{sec:related}
% =====================================================================

\subsection{Model Compression}

Quantization reduces weight precision from 32-bit floating point to 8-bit integers or lower~\citep{jacob2018quantization, gholami2022survey}. Post-training quantization is simple but tends to damage accuracy on medical images where the signal lives in subtle color gradients. QAT simulates quantization during training so the network learns to tolerate the noise~\citep{tfmot2023}. Jacob et al.~\citep{jacob2018quantization} reported near-lossless INT8 inference on ImageNet, though medical tasks with extreme class imbalance and fine spatial features fail differently.

Knowledge distillation trains a compact student network to match softened probability outputs from a larger teacher~\citep{hinton2015distilling}. The temperature parameter controls how much of the teacher's internal ranking over classes leaks through the soft targets. Feature-level distillation, matching intermediate layer representations through MSE loss, outperforms logit-only distillation on segmentation where spatial structure matters at every pixel~\citep{romero2015fitnets}.

Pruning~\citep{han2015learning} and neural architecture search are complementary approaches I did not evaluate here. Both could be stacked on top of QAT and KD, and I plan to investigate that combination in follow-up work. Choudhary et al.~\citep{choudhary2020comprehensive} provide a broad survey of compression methods for reference.

\subsection{Efficient Attention}

Standard self-attention is quadratic in sequence length. For a Vision Transformer~\citep{dosovitskiy2021image} processing a 224$\times$224 image at patch size 16, that is 196 tokens attending to each other at every layer. Linformer~\citep{wang2020linformer} projects to lower dimensions. Performer~\citep{choromanski2021performers} uses random feature approximations. FlashAttention~\citep{dao2022flashattention} keeps exact attention but restructures memory access patterns.

Chen et al.~\citep{chen2026msa} took a different route with Memory Sparse Attention (MSA). They compress the KV cache by averaging contiguous token chunks (chunk-mean pooling) and route each query to only the top-$k$ most relevant compressed chunks. On language benchmarks, MSA scaled to 100 million tokens with under 9\% degradation. The 94.84\% accuracy on needle-in-a-haystack retrieval at 1M tokens is what originally caught my eye---it suggested the mechanism could handle spatial redundancy in images too.

\subsection{Medical Imaging on Consumer Hardware}

Prior work on compressing medical models has targeted mobile phones~\citep{pacheco2024compressed} and edge devices~\citep{qayyum2023efficient}. The endpoint scenario differs: workstations have 8--32 GB of RAM and multi-core CPUs but no GPUs. The binding constraint shifts from raw model size to CPU inference latency. A 10 MB model is fine. 300 ms per image is not.

TensorFlow Lite~\citep{tflite2023} and ONNX Runtime~\citep{onnxruntime2023} both handle CPU inference on macOS, Windows, and Linux. They are production-ready. What nobody had done was evaluate compressed medical models on these runtimes with endpoint deployment constraints---model size under 10 MB, latency under 50 ms, RAM under 100 MB.

\section{Methods}
\label{sec:methods}
% =====================================================================

\subsection{Datasets}

\textbf{ISIC 2020.} 33,126 dermoscopy images for binary melanoma classification~\citep{codella2019isic}. Melanoma prevalence sits around 2\%, so the dataset is heavily lopsided. I resized images to 224$\times$224, normalized to $[-1, 1]$, and used inverse-frequency class weighting. Augmentation included random flips, brightness ($\pm$0.2), and contrast ($\pm$0.2). I split 70/15/15 for train/validation/test with stratification on the target label.

\textbf{BraTS 2021.} Multi-modal brain MRI volumes (T1, T1ce, T2, FLAIR) with four segmentation classes: background, necrotic core, peritumoral edema, enhancing tumor~\citep{baid2021brats}. I remapped labels from $\{0,1,2,4\}$ to $\{0,1,2,3\}$ for standard one-hot encoding and used a 2.5D approach: 3 adjacent axial slices stacked across 4 modalities produce a 12-channel 128$\times$128 input. This gives some volumetric context without the 3D convolutions that would make CPU inference impractical. I filtered background-only slices and z-score normalized each volume independently.

\subsection{Baseline Models}

For classification I used EfficientNetB0~\citep{tan2019efficientnet} pretrained on ImageNet with a custom head: global average pooling, batch normalization, dropout (0.3), a 256-unit dense layer with ReLU, dropout (0.15), and a sigmoid output. I unfroze the last 20 backbone layers for domain adaptation and trained with binary cross-entropy, Adam at $10^{-4}$, and early stopping on validation AUC (patience 7 epochs).

For segmentation I implemented two U-Net~\citep{ronneberger2015unet} variants. The full model has four encoder stages with [64, 128, 256, 512, 1024] filters and is the teacher. The lite model has three stages with [32, 64, 128, 256] filters---about 8$\times$ fewer parameters---and is the student. Both use combined Dice + cross-entropy loss, Adam at $10^{-3}$, with early stopping on validation Dice.

\subsection{Quantization-Aware Training}

I used the TensorFlow Model Optimization Toolkit~\citep{tfmot2023} to wrap trained baselines with fake-quantization nodes that simulate INT8 arithmetic during forward passes. Fine-tuning ran 10 epochs at $10^{-5}$ learning rate. After that I stripped the QAT wrappers and exported to TFLite with full INT8 quantization. The INT8 conversion needs a representative calibration dataset; I used 200 randomly sampled training images. I also exported FP16 variants for comparison.

\subsection{Knowledge Distillation}

For classification the teacher was EfficientNetB3 (12M parameters) and the student was MobileNetV3-Small~\citep{howard2019mobilenetv3} (2.5M parameters). For segmentation the teacher was the full U-Net and the student was the lite U-Net.

The distillation loss combines a soft-target KL divergence with a hard-target cross-entropy:
\begin{equation}
\mathcal{L} = \alpha \cdot \text{KL}\!\left(\sigma\!\left(\frac{z_t}{T}\right) \middle\| \sigma\!\left(\frac{z_s}{T}\right)\right) + (1-\alpha) \cdot \text{CE}(y, z_s)
\label{eq:kd_loss}
\end{equation}
where $z_t$ and $z_s$ are teacher and student logits, $T$ is temperature, $\alpha$ weights the two terms, and $\sigma$ is softmax (sigmoid for binary classification). I used $T{=}4.0$, $\alpha{=}0.7$ for classification and $T{=}3.0$, $\alpha{=}0.6$ for segmentation. For segmentation I added feature-level MSE distillation with 1$\times$1 conv adapters for dimension matching, following~\citet{romero2015fitnets}.

\subsection{Sparse Attention Compression}

I adapted two mechanisms from MSA~\citep{chen2026msa} for the classification pipeline.

\textbf{KV cache pooling.} Key and value tensors get averaged in contiguous chunks of size $p$ along the spatial dimension. A kernel of 4 compresses 196 spatial tokens (14$\times$14 patch grid from 224$\times$224 input at patch size 16) down to 49 pooled tokens. This mirrors the \texttt{sequence\_pooling\_kv} operation in MSA's codebase.

\textbf{Top-$k$ routing.} I compute scaled dot-product scores between queries and pooled keys, reduce across heads (max pooling) and query positions (max pooling), then select the top-$k$ chunks. Attention runs only over those $k$ chunks. With $k{=}8$ out of 49, that is a further 6$\times$ reduction on top of the 4$\times$ from pooling. The theoretical combined reduction is 24$\times$ in attention memory.

\subsection{Export and Evaluation}

All compressed models export to TFLite (INT8, FP16, FP32) and ONNX. I measured:
\begin{itemize}
    \item \textbf{Classification:} ROC-AUC on the ISIC test set.
    \item \textbf{Segmentation:} Mean Dice coefficient excluding background on the BraTS test set.
    \item \textbf{Model size:} File size of the exported model in megabytes.
    \item \textbf{Latency:} Median and $p_{95}$ wall-clock time per sample over 100 CPU runs via the TFLite interpreter.
\end{itemize}

\subsection{Sparse Attention Compression}
\label{sec:sparse_results}

Table~\ref{tab:sparse} and Figure~\ref{fig:sparse_ablation} show the sparse attention results. Kernel=4, top-$k$=8 reduces attention memory 24.5$\times$ at 0.7\% AUC cost. I did not expect this to work as well as it did. Adjacent dermoscopy patches carry the same color and texture most of the time, so averaging groups of four barely touches the signal that matters for classification. The top-$k$ router learns to attend to the lesion and ignore uniform skin---the same thing a dermatologist does, in hindsight.

At kernel=8 with top-$k$=4, accuracy drops 3.4\%. That is too aggressive. At 14$\times$14 resolution, a kernel of 8 averages half a row of patches, which is too coarse for lesion boundaries that can cut across the grid at arbitrary angles. A 2D pooling kernel that respects the spatial layout would probably fix this, but I have not tried it yet.

\section{Discussion}
\label{sec:discussion}
% =====================================================================

\subsection{Deployment as Endpoint Software}

A 2.7 MB melanoma classifier that runs in 8 ms fits in a system tray app. No cloud calls, no internet, no patient data leaving the machine. I have pushed hundreds of packages through Jamf and Intune over the years, and a TFLite model is just another file in the payload. Version it alongside the application, roll back by swapping the binary. Nothing about this needs new infrastructure.

The MedCompress desktop application (\texttt{deploy/app.py}) wraps this into a working GUI: select an image, click analyze, see the result. The CLI variant (\texttt{deploy/cli.py}) processes batches and outputs JSON for integration with existing clinical workflows. Both run on macOS, Windows, and Linux without a GPU.

\subsection{Where Compression Fails}

Segmentation takes more damage from quantization than classification does. INT8 discretizes activations into 256 levels, and when the decision boundary between tumor and healthy tissue spans only a few pixels, that discretization shifts predictions by enough to matter. FP16 loses 0.3\% Dice versus INT8's 2.2\%, so for clinical segmentation where boundary accuracy is non-negotiable, the 2$\times$ larger FP16 file is the safer option.

Distillation degrades segmentation because the lite U-Net has one fewer encoder stage. Three stages instead of four means lower spatial resolution at the bottleneck. Some of the teacher's spatial detail cannot fit in the student's representation, no matter how good the distillation loss is.

\subsection{Sparse Attention: From Language Models to Medical Vision}

MSA~\citep{chen2026msa} pools along a 1D token sequence where documents are semantically distinct. Image patches live on a 2D grid where a melanoma border can cut across chunk boundaries at any angle. That is a real mismatch. At kernel=4 the mismatch is tolerable because most patches within a group share the same texture. At kernel=8 it stops being tolerable---the model averages over half a row of the 14$\times$14 grid, losing the fine spatial detail needed for lesion detection. A 2D-aware pooling kernel that aligns with the patch grid would be the natural next step, but I have not built it yet.

\subsection{Limitations}

I evaluated two datasets and two tasks. Chest X-ray classification, retinal imaging, and histopathology would test whether these compression ratios generalize. The latency numbers come from controlled benchmarks, not machines running clinical software at the same time. I did not try pruning or NAS. Sparse attention only applies to classification here; segmentation tasks where every spatial position carries weight need a different approach. I have not validated compressed predictions against pathologist ground truth in a deployment setting.

\subsection{Future Directions}

My near-term plan is to add chest X-ray and retinal classification, test real endpoint performance under concurrent clinical workloads, and build a 2D pooling kernel for spatial-aware sparse attention. I also want to add uncertainty quantification---compressed models sit closer to their accuracy floor, and clinicians need to know when a prediction is unreliable.

The deployment architecture I am working toward is a lightweight daemon running on managed endpoints, processing medical images as they arrive. The compression ratios and latency numbers in this paper say that should work. The open-source application in \texttt{deploy/} is the first version, and it runs, which is more than I could say when I started.

\section{Conclusion}
\label{sec:conclusion}
% =====================================================================

Medical imaging models compress to single-digit megabyte sizes and sub-20 ms CPU inference times without losing clinical usefulness. QAT and KD together give 6$\times$ compression on classification and 30$\times$ on segmentation. Sparse attention adapted from MSA~\citep{chen2026msa} is a complementary path for attention layers. The code, export pipelines, and desktop application are open source.

The Mac on a clinic desk can run medical AI. The models just need to be smaller. This paper is my attempt at figuring out how small is small enough.
